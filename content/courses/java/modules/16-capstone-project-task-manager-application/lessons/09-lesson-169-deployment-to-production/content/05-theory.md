BOTH PATHS

Fly.io is a modern platform that makes deploying full-stack applications simple. Here is a step-by-step guide to deploying our Task Manager.

**Thymeleaf path advantage:** You only need ONE Fly.io app (Spring Boot with embedded frontend). Your entire application deploys as a single container -- no separate frontend deployment needed.

**React path:** You need TWO Fly.io apps (Spring Boot API + React frontend served by nginx).

Step 1: Create Fly.io Account
- Go to https://fly.io
- Sign up with GitHub (recommended for easy integration)
- No credit card required for the free tier

Step 2: Install Fly.io CLI and Login
```bash
# Install Fly.io CLI (flyctl)
# See: https://fly.io/docs/hands-on/install-flyctl/

# For macOS:
brew install flyctl

# For Linux:
curl -L https://fly.io/install.sh | sh

# Login
fly auth login
```

Step 3: Launch Your App
```bash
# Navigate to your project root
cd taskmanager

# Launch creates fly.toml config file
fly launch --no-deploy

# Answer the prompts:
# - App name: taskmanager-app (or choose unique name)
# - Region: pick closest to you
# - PostgreSQL: Yes (creates managed database)
# - Redis: No (unless you need caching)
```

Step 4: Configure Environment Variables
```bash
# Set production profile
fly secrets set SPRING_PROFILES_ACTIVE=production

# Set JWT secret (generate a secure random string)
fly secrets set JWT_SECRET=your-secure-random-string-here

# Database URL is automatically set by Fly.io when you create PostgreSQL
# Verify with: fly secrets list
```

Step 5: Create fly.toml (auto-generated by launch, customize if needed):
```toml
app = 'taskmanager-app'
primary_region = 'iad'

[build]
  dockerfile = 'Dockerfile'

[http_service]
  internal_port = 8080
  force_https = true
  auto_stop_machines = 'stop'
  auto_start_machines = true
  min_machines_running = 0
  processes = ['app']

[[vm]]
  size = 'shared-cpu-1x'
  memory = '256mb'
```

Step 6: Deploy
```bash
# Deploy your application
fly deploy

# View logs
fly logs

# Check status
fly status

# Open your deployed app
fly open
```

Step 7: Deploy Frontend (React path only)
```bash
cd frontend

# Launch separate frontend app
fly launch --no-deploy

# Set API URL pointing to your backend
fly secrets set VITE_API_URL=https://taskmanager-api.fly.dev

# Deploy
fly deploy
```

Step 8: Set Up Custom Domain (Optional)
```bash
# Add custom domain
fly certs add your-domain.com

# Configure DNS with CNAME record pointing to your-app.fly.dev
# Fly.io automatically provisions SSL certificates
```

Fly.io Dashboard Features:
- Real-time logs and metrics
- Automatic SSL certificates
- Environment variable/secret management
- Usage analytics
- One-click rollbacks
- PostgreSQL database management

Free tier limits:
- 256MB RAM per machine
- 3GB transfer per month
- Shared CPU
- Up to 3 machines running concurrently
